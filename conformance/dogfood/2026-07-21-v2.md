# Source-Blind V2 Probes — 2026-07-21

Contract: [`../expected-behavior.md`](../expected-behavior.md)

## Method

One fresh, non-persistent Claude Code session received only the 1,400-word V2 protocol and five synthetic scenarios. Safe mode was enabled, all tools were disabled, and the evaluator had no checker source, diff, tests, repository history, DACS context, ISA, or Beads state. Runtime `modelUsage` recorded `claude-sonnet-5`; a small classifier call used Haiku.

The CLI was then exercised separately from an isolated temporary directory through its documented command surface. Inputs were the claim-only starter and the optional Work Graph example; no implementation source was inspected during that operator probe.

## Scenario results

- **Greenfield serial:** pass. Created one missing product authority, selected `frontier_kind: claim`, and created no ledger for two short serial claims.
- **Brownfield:** pass. Preserved the adequate existing `PRODUCT.md` authority and created no competing spec or ledger.
- **External gate:** pass. Kept the human-gated release claim outside the frontier while selecting the independent local diagnostics claim.
- **Parallel workers:** pass. Selected `frontier_kind: vertical` only with explicit owners, disjoint file and runtime scopes, no unfinished cross-dependency, and one reconciliation owner.
- **Review reopen:** pass. Reopened the verified claim only after the finding was accepted and reproduced at its snapshot; retained prior evidence and allowed verified progress to decrease.

## CLI observations

`graph-check starter/SPEC.md --json` returned a valid claim-only report with two claims, zero slices, `frontierKind: "claim"`, and `activeFrontier` equal to `claimFrontier`.

`graph-check examples/notes-cli/SPEC.md --json` returned a valid scaled report with five claims, three slices, `frontierKind: "vertical"`, `claimFrontier: ["C-2", "C-4"]`, `activeFrontier: ["S-2"]`, and `blocked: ["S-3"]`.

## Source-aware review disposition

A separate read-only Fable pass inspected the real V2 diff. The Claude result recorded only `claude-fable-5` in `modelUsage`; no Opus fallback occurred. Four findings were accepted: fail-closed handling for misplaced Work Graph markers, cleaner unowned-parallel diagnostics, a complete START report field set, and an expanded first use of Ideal State Artifact. Regression tests accompany the checker fixes.

Codex Autoreview then reported three P2 findings. Two were reproduced and fixed: misplaced execution markers could bypass validation, and unrestricted wildcard scopes could overstate parallel isolation. The third was rejected after direct code inspection and new regression coverage: blocked and unknown owned claims were already excluded by `claimsInternallyClosed` and classified in the blocked vertical bucket. The final Autoreview pass was clean.

## Limits

These probes test whether the protocol produces the intended authority, frontier, scaling, and reopening decisions in bounded scenarios. They do not establish universal productivity, autonomous correctness, or release certification.
